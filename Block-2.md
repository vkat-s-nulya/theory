# ООП

## Общие вопросы

### 1. Что такое ООП?

Обьектно Ориентированное Программирование - парадигма, в которой программа строится из обьектов

### 2. Что такое класс?

Пользовательский тип данных, который описывает структуру и поведение обьекта

### 3. Что такое объект?

Экземпляр класса

### 4. В чем отличие между классом и структурой?

В модификаторе доступа по умолчанию, у структуры private, у класса private

### 5. Расскажи про основные принципы ООП (абстракция, наследование, полиморфизм, инкапсуляция)

- Абстракция: выделение существенного и скрытие несущественного, то есть снаружи виден только интерфейс взаимодействия, а реализация скрыта
- Инкапсуляция: сокрытие внутренного состояния обьекта
- Наследование: создание новых классов на основе существующих
- Полиморфизм: возможность работать с разными типами данных через единый интерфейс

### 6. Что такое конструктор?

Специальный метод класса, который вызывается при создании обьекта

### 7. Существует ли виртуальный конструктор?

Нет, потому что таблица вирт функций храниться внутри обьекта + виртуализация нужна если неизвестен точный тип, а при создании обьекта он гарантировано должен быть известен

### 8. Какие виды конструкторов вы знаете?

- По умолчанию: без параметров
- Параметризированный: принимает аргументы
- Копирования: создает обьект как копию другого
- Перемещения: берет временный обьект вместо копирования
- Преобразования: конструктор с одним параметром, который неявно преобразовывает этот параметр к типу
- Делегирующий: один конструктор вызывает другой

### 9. Что такое деструктор?

Специальный метод класса, который автоматически вызывается при уничтожении обьекта, для освобождения ресурсов

### 10. В чем отличие АБК от интерфейса?

АБК - класс, у которого есть хотя бы один чисто виртуальный метод (= 0)
Может содержать вирт методы, поля, конструкторы
Интерфейс - частный случай АБК, содержит исключительно чисто вирт функции, нет полей, нет реализаций

### 11. Расскажи про принципы SOLID.

- S - Single responsibility
класс должен отвечать только за одну логическую вещь
- O - Open/Closed
класс открыты для расщирения, но закрыты для модификации
- L - Liskov substitution 
Обьекты дочернего класса должны подменять обьекты родительского без нарушения корректности
- I - Interface segregation 
Клиенты не должны зависеть от методов, которыми не пользуются
- D - Dependency inversion
Модули верхнего уровня не зависят от модулей нижнего, оба должны зависить от абстракции

### 12. Расскажи, как устроена лямбда.

Лямбда функция - анонимная функция, которую можно определить прямо в место использования. Под капотом компилятор генерирует класс с перегруженным оператором (), а захваченные переменные становятся полями этого класса

### 13. Сколько памяти занимает пустой класс?

1 байт, по стандарту гарантируется что 0 не бывает, а то обьекты были бы не различимы по адрессу, что нарушает фундаментальное правило

## Ключевые слова

### 14. Зачем используется virtual?

Для обьявления, что метод может быть переопределен в наследниках

### 15. Зачем используется override?

Для обьявление, что метод переопределяет виртуальный метод базового класса

### 16. Зачем используется mutable (для полей класса, для лямбды)?

Для полей класса: для разрешения конст методам менять поля класса
Для лямбды: для возможности менять захваченные по значению переменные внутри лямбды

### 17. Зачем используется final?

Для запрета дальнейшего наследования для класса или переопределения для метода

### 18. Зачем используется explicit?

Для запрета неявного преобразования через конструктор или приведения типа

### 19. Зачем используется noexcept?

Для обьявления что функция не бросает исключений

## Методы класса

### 20. Что такое виртуальная функция?

Метод класса, для которого реализация выбирается в рантайме по фактическому типу обьекта, а не по типу указателя или ссылки

### 21. Зачем нужен виртуальный деструктор?

Чтобы при удалении обьекта через указатель на базовый класс вызывался деструктор реального типа а не только базового. Деструкторы вызываются вверх по иерархии

### 22. В чем особенность константных методов класса?

Метод обещает не менять состояние обьекта

### 23. Что такое чистая виртуальная функция?

Виртуальный метод без реализации, обьявленный с = 0 в конце, наследники обязаны его реализовать

### 24. Как получить ошибку pure virtual call? Напиши код.

Рантайм ошибка, возникающая при попытке вызвать чисто виртуальную функцию, у которой еще нет реализации. В основном это происходит в вызове метода из конструктора/деструктора базового класса

```
class Base
{
public:
    Base()
    {
        init();
    }

    virtual void init() = 0;
    virtual ~Base() = default;
};

class Derived : public Base
{
public: 
    void init() override
    {
        std::cout << "Derived";
    }
}

int main()
{
    Derived d;
    return 0;
}
```

### 25. Расскажи Правило 5 (Правило 3, Правило 0). Напиши код.

Правило о том, какие спец методы нужно определять в классе

Rule of 3 - если определяется деструктор, конструктор копирования или оператор копирующего присваивания - нужно определять все три. Явный признак, что класс управляет ресурсом и могут возникуть shared state или double free
Rule of 5 - эволюция rule of 3 после с++11, добавились move конструктор и move присваивание
Rule of 0 - лучший класс тот, кто не определяет ни одной из 5. Это значит, что вместо ручного управление ресурсом используется что-то что уже управляет по rule of 5

```
class Buffer
{
private:
    int* data;
    size_t size;
public:
    Buffer(size_t n) : data(new int[n]), size(n) {}
    ~Buffer() { delete[] data; }
    Buffer(const Buffer& buffer) : data(new int[buffer.size]), size(buffer.size)
    {
        std::copy(buffer.data, buffer.data + buffer.size, data);
    }

    Buffer& operator=(const Buffer& buffer)
    {
        if (this == &buffer) return *this;
        int* tmp = new int[buffer.size];
        std::copy(buffer.data, buffer.data + buffer.size, tmp);
        delete[] data;
        data = tmp;
        size = buffer.size;
        return *this;
    }

    Buffer(Buffer&& buffer) noexcept : data(buffer.data), size(buffer.size)
    {
        buffer.data = nullptr;
        buffer.size = 0;
    }

    Buffer& operator=(Buffer&& buffer) noexcept
    {
        if (this == &buffer) return *this;
        delete data[];
        data = buffer.data;
        size = buffer.size;
        buffer.data = nullptr;
        buffer.size = 0;
        return *this;
    }
}
```

### 26. Расскажи об использовании static для полей и методов  класса.

static означает что поле или метод принадлежит классу, а не конкретному обьекту. Доступен без создания обьекта и живет в одном экземпляре на всю программу

### 27. Расскажи об использовании inline для полей и методов класса.

Позволяет иметь определение метода прям в заголовке и при включении в несколько единиц трансляции ошибок не будет. Если определение написано прям в заголовке то inline будет нявный. 
С с++17 можно обьявить inline static переменную и сразу же в заголовке ее определить

### 28. Что такое vtable? Расскажи, как она работает.

Специальный механизм, который реализует динамический полиморфизм.
Как устроено:
1) Компилятор создает для каждого класса с вирт методами массив указателей на методы - по одному на каждый вирт метод. Это таблица вирт функций 
2) В каждый класс с вирт методами через поле добавляется указатель на эту таблицу. Это делается конструктором
3) Превращает все виртуальные вызовы просто в косвенное обращение к функции найдя ее в vtable

## Шаблоны классов

### 29. Что такое шаблонный класс?

Инструкция для генерации класса, параметризованный одним или несколькими типами/значениями

### 30. Может ли конструктор быть шаблонной функцией?

Да, но нельзя явно указать в <>

### 31. Можно ли объявить виртуальный метод класса шаблонным?

Нет, стандарт это явно запрещает. Это связано с тем размер vtable обязан быть известен в компайл тайм, а если есть виртуальный метод класса шаблонный, то он неизвестен

### 32. Можно ли объявить метод шаблонного класса виртуальным?

Да, потому что vtable будет создана фиксированного размера для каждого типа

### 33. Можно ли наследоваться от шаблонного класса?

Да, и есть несколько вариантов:
- наследник фиксирует шаблонный параметр
- шаблонный наследник шаблонного класса
- наследник фиксирует часть параметров 
- наследник передает себя базовому классу как шаблонный параметр (CRTP)

### 34. Что такое SFINAE. Напиши код

Правило, говорящее о том что если в процессе подстановки шаблонного параметра в сигнатуру получается невалидный тип, то ошибка не бросается, а компилятор просто пойдет искать другую подстановку

```
template <typename T>
void f(typename T::type) {}

template <typename T>
void f(...) {}

struct A
{
    using type = int;
}

f<A>(0);
f<int>(0);
```

## Семантика перемещения

### 35. Что такое семантика перемещения?

Механизм, который позволяет передавать ресурсы обьекта без копирования. Вместо создания нового обьекта мы просто перенесем ресурсы одного обьекта в другой

### 36. Что такое deep copy и shallow copy?

shallow copy - копирование указателей, то есть два обьекта ссылаются на одну и ту же память 
deep copy - полное копирование данных с выделением новой памяти

### 37. Что такое rvalue и lvalue? Какие еще категории значений вы знаете?

lvalue - обьект с именем 
rvalue - временный обьект, который нельзя использовать повторно
также есть prvalue, xvalue, glvalue

### 38. Что делает std::move? Как устроен std::move?

std::move - преобразует обьект в rvalue ссылку. Сам по себе ничего не перемещает а лишь делает статик каст к T&&

### 39. Что делает std::forward? Как устроен std::forward?

std::forward - используется для сохранения категории значения в шаблонах, то есть lvalue оставит как lvalue и rvalue оставит как rvalue 

## Исключения

### 40. Что такое исключение? Расскажи, как работает механизм исключений.

Это механизм обработки ошибок, при котором управление переходит из места возникновения ошибки в ее обработчик
При вызове throw происходит раскрутка стека, во время которой вызываютсч деструкторы всех созданных локальных обьектов. После этого ищется подходящий catch блок, если не нашелся - std::terminate

### 41. Когда нужно использоваться исключения, а когда коды ошибок?

Если совсем просто: коды ошибок применяются в ожидаемых ситуациях, исключения для неожидаемых
По факту коды ошибок используют в низкоуровневом коде или для повышения производительности, в остальных случаях обычно пользуются исключениями

### 42. Расскажите о иерархии расположения catch-блоков.

Проверка типа идет сверху вниз, более специфичные типы стоит писать выше, иначе они никогда не будут достигнуты, в конце надо ставить catch(...), который примет что угодно 

### 43. Можно ли бросать исключение из конструктора? О чем нужно помнить, когда мы бросаем исключения из конструктора?

Да, тогда обьект считается не созданным -> деструктор не вызывется, однако уже созданные поля будут корректно уничтожены, важно помнить о ресурсах выделенных перед throw и не являющийся полями класса

### 44. Можно ли бросать исключение из деструктора?

Можно, но не нужно. Если при раскрутке стека и вызове декструктора бросится исключения то вызовется std::terminate, по умолчанию все декструкторы noexcept

### 45. Что произойдет, если при попытке выделить память с помощью new памяти не хватит?

Бросится исключение std::bad_alloc 

### 46. Как сделать так, чтобы new не бросал исключение?

написать new(std::nothrow), тогда вернется nullptr и мы сами что-то делаем на основе этого

### 47. Какие виды кастов в С++ вы знаете? Расскажите про каждый из них (не забудь про C-style cast).

static_cast<T>(obj) преобразование на этапе компиляции
dynamic_cast<T>(obj) преобразование в рантайме
const_cast<T>(obj) меняет cv-классификаторы
reinterpret_cast<T>(obj) побитовое преобразование без гарантий
(T)obj С-стиль каста, который включает в себя все эти касты

### 48. Что произойдет, если std::dynamic_cast не сможет скастовать тип?

если указатель - вернется nullptr 
еслт ссылка - бросается std::bad_cast

## Идиомы

### 49. Что такое идиома RAII? Напиши код. Приведите примеры использования RAII из STL.

Resource acquisition is initialization
Привязка времени жизни ресурса к времени жизни обьекта, то есть ресурс захватывается в конструкторе и освобождается в деструкторе
самый простой пример std::unique_ptr

```
class IntArray
{
private:
    int* data;
    size_t size;
public:
    IntArray(size_t n) : data(new int[n]), size(n) {}
    ~IntArray() { delete[] data; }
}

void f()
{
    IntArray arr(100); // захватили ресурс
} // освободили ресурс
```

### 50. Что такое идиома PIMPL? Напиши код.

Pointer to IMPLementation 
все приватные детали класса скрываются в отдельной структуре, а в основном классе только указатель на эту структуру

```
// user.h

class User
{
public:
    User(const std::string& name);
    ~User();
    void print() const;
private:
    struct Impl;
    std::unique_ptr<Impl> pImpl;
}

// user.cpp

struct User::Impl
{
    std::string name;
}

User::~User() = defualt; // важно именно тут, иначе компилятор не найдет реализацию User::Impl и не скомпилится

void User::print() const
{
    std::cout << pImpl->name;
}
```

### 51. Что такое RTTI? Напиши код.

Run-Time type information
механизм позволяющий получать информацию о типе обьекта в рантайме
работает только для полиморфных классов, нужно для typeid и dynamic_cast

```
class Animal
{
public:
    virtual ~Animal() = default;
};

class Dog : public Animal {};
class Cat : public Animal {};

int main()
{
    Animal* a = new Dog();
    std::cout << typeid(*a).name();

    if (Dog* dog = dynamic_cast<Dog*>(a))
        // зайдем

    if (Cat* c = dynamic_cast<Cat*>(a))
        // не зайдем
}
```

### 52. Что такое CRTP? Напиши код.

Curiosly Recurring Template Pattern
Паттерн, в котором класс наследуется от шаблонной базы, передавая самого себя как параметр шаблона

```
template <typename Derived>
class Base
{
public:
    void base() 
    { 
        static_cast<Derived*>(this)->derived();
    }
};

class Dog : public Base<Dog>
{
public:
    void derived() { std::cout << "dog"; }
};

class Cat : public Base<Cat>
{
public:
    void derived() { std::cout << "cat"; }
};

int main()
{
    Dog dog;
    dog.base(); // dog

    Cat cat;
    cat.base(); // cat
}
```

### 53. Что такое CAS (гарантия безопасности при копировании объектов)? Напиши код.

Compare and swap
Основная идея это сначала создать копию и только потом менять старый обьект с копией. Это позволяет сделать так что если на этапе копирования возникла ошибка, то исходный обьект не изменился и ошибка не оставит обьект в непонятном состоянии. Как следствие это дает strong гарантии на исключения

```
class Buffer
{
private:
    int* data;
    size_t size;

public:
    Buffer(size_t n) : data(new int[n]), size(n) {}
    ~Buffer() { delete[] data; }

    // copy ctor - создаст копию
    Buffer(const Buffer& other) : data(new int[other.size]), size(other.size) 
    {
        std::copy(other.data, other.data + size, data);
    }

    // swap который не вызовет исключение
    friend void swap(Buffer& a, Buffer& b) noexcept
    {
        std::swap(a.data, b.data);
        std::swap(a.size, b.size);
    }

    // CaS
    Buffer& operator=(Buffer other) // создалась копия 
    {
        swap(*this, other);         // меняемся с копией
        return *this;               // старая data погибает в деструкторе other
    }
}
```

### 54. Что такое RVO? Напиши код.

Return value optimization
оптимизация компилятора, которая убирает создание временного обьекта при возврате из функции, то есть обьект сконструируется сразу в месте назначения. Обязательно по стандарту

```
class A
{
public:
    A()         { std::cout << "ctor"; }
    A(const A&) { std::cout << "copy"; }
    A(A&&)      { std::cout << "move"; }
    ~A()        { std::cout << "dtor";}
}

A create() 
{
    return A();
}

int main()
{
    A a = create();
    // без RVO
    // ctor -> move -> dtor -> move -> dtor
    // с RVO
    // ctor -> dtor
}
```
### 55. Что такое NRVO? Напиши код.

Named return value optimization
Частный случай RVO, компилятор устраняет перемещение для именованного локального обьекта, возвращаемого из функции. Не обязательно по стандарту

```
class A
{
public:
    A()         { std::cout << "ctor"; }
    A(const A&) { std::cout << "copy"; }
    A(A&&)      { std::cout << "move"; }
    ~A()        { std::cout << "dtor";}
}

A create() 
{
    A a;
    return a;
}

int main()
{
    A a = create();
    // без NRVO
    // ctor -> move -> dtor -> move -> dtor
    // с NRVO
    // ctor -> dtor
}
```

### 56. Что такое SSO?

Small string optimization
Оптимизация, при которой короткие строки хранятся на стеке, а не в куче. У нас занято немного места под capacity size, так что их просто переиспользуем

### 57. Что такое EBO? Напиши код.

Empty Base Optimization
оптимизация компилятора, при которой базовый класс не занимает места в наследнике. Любой обьект обязан иметь ненулевой размер (в нашем случае пустой базовый класс), так что вот эта оптимизация убирает этот лишний байт из наследника

```
class Empty {};

class Derived : Empty
{
    int x;
}

sizeof(Derived); // 4, то есть только int
```
